1st Project Update: Learning the basics/ fundamentals of coding a Blockchain

A blockchain by design should be in a sense an immutable decentralized network which is able to keep track of things like transactions for example on a ledger. For this 1st project update I will try to explain what I have learnt in terms of the fundamentals of building a blockchain on Python. A blockchain is secured using a hash system which stores the information for the given block securely. This is secure because if anything within the blockchain is changed the hash will change drastically thus the network can automatically determine where the inconsistent transaction occurred and as a result deny it.

I will be using Python for this project as I am more comfortable with the language as it is much more simplistic compared to other languages. First of all we have to import the hashlib library as it will be used to generate the hash for each block later on. Moving on we have to define our block as a class. The first part is the initializer which requires the input of the previous hash and the transactions that will be stored in this current block. Next up we have to construct a data string which is called block data in order to store the transactions as a string. On top of that we also create a block hash to store the hash code generated by sha-256 for the unique data stored in the given block.

As this is just a fundamental look at the basics of building a blockchain I will populate the transactions here myself. By right, the transactions should be populated in the network and cross-checked by a network of miners.
